        通过缓存的方式暂存计算结果，这样就可以用缓存的计算结果代理重复的需要计算的地方从而提高性能

    2、$.proxy

        通过代理的方式：将上下文对象绑定到需要代理的函数中去

    3、ES6的 Proxy

    4、事件委托

        可以将事件全部委托给父节点进行代理处理

    5、图片懒加载

        让图片懒加载变的更加符合：职责单一、开闭原则的设计原则

        图片懒加载的行为、通过代理的方式实现懒加载的功能

    6、防抖代理

        可以通过代理行为从而实现防抖节流的操作

    7、代理跨域

        通过配置代理服务器从而实现跨域
